## 프로젝트 개요

### 목적
학교에서 학년별 수업 계획서를 웹페이지에서 직접 작성하고, 저장하며, 다른 교사들도 열람할 수 있는 시스템입니다.

### 기술 스택
- **HTML**: 웹페이지 구조
- **CSS**: 디자인 및 스타일링
- **JavaScript**: 동적 기능 및 데이터 관리
- **LocalStorage**: 브라우저 로컬 저장소를 이용한 데이터 저장

### 주요 특징
- 파일 다운로드 없이 웹페이지 내에서 작성 및 저장
- 초등학교(1-6학년), 중학교(1-3학년), 고등학교(1-3학년) 구분
- 비밀번호 보호 기능으로 본인만 수정 가능
- 반응형 디자인으로 모바일/태블릿에서도 사용 가능

## 파일 구조

```
프로젝트 폴더/
│
├── index.html          # 메인 HTML 파일
├── styles.css          # 스타일시트 파일
├── script.js           # JavaScript 기능 파일
└── 구현설명서.md       # 이 문서
```

### 각 파일의 역할

1. **index.html**
   - 웹페이지의 구조를 정의
   - 초등/중등/고등학교 탭 및 학년별 섹션
   - 계획서 작성/보기/수정을 위한 모달창

2. **styles.css**
   - 웹페이지의 디자인과 레이아웃
   - 색상, 글꼴, 버튼 스타일 등
   - 반응형 디자인 (모바일 대응)

3. **script.js**
   - 모든 동적 기능 구현
   - 데이터 저장/불러오기/수정/삭제
   - 비밀번호 확인 기능

## 주요 기능 설명

### 1. 학교별 탭 전환
- 초등학교, 중학교, 고등학교 탭을 클릭하여 전환
- 각 학교별로 학년이 자동으로 구분됨

### 2. 계획서 작성
- 각 학년의 "+ 계획서 작성" 버튼 클릭
- 필수 정보 입력:
  - 교사 이름
  - 과목
  - 계획서 제목
  - 계획서 내용
  - 비밀번호 (수정 시 필요)

### 3. 계획서 보기
- 작성된 계획서를 클릭하면 상세 내용 확인
- 작성일 및 최종 수정일 표시

### 4. 계획서 수정
- 계획서 상세보기에서 "수정하기" 버튼 클릭
- 작성 시 입력한 비밀번호 입력
- 비밀번호가 일치하면 수정 가능

### 5. 계획서 삭제
- 계획서 상세보기에서 "삭제하기" 버튼 클릭
- 비밀번호 확인 후 삭제

#### 2. 탭 전환 기능
```javascript
function initializeTabs() {
    tabButtons.forEach(button => {
        button.addEventListener('click', function() {
            // 모든 탭 비활성화
            tabButtons.forEach(btn => btn.classList.remove('active'));
            // 선택된 탭 활성화
            this.classList.add('active');
            const school = this.getAttribute('data-school');
            document.getElementById(school).classList.add('active');
        });
    });
}
```
- 클릭한 탭을 활성화하고 해당 학교 섹션 표시

#### 3. 계획서 저장
```javascript
function savePlan() {
    const plan = {
        id: generateId(),
        school: currentSchool,
        grade: currentGrade,
        teacherName: teacherName,
        subject: subject,
        title: title,
        content: content,
        password: password,
        createdAt: new Date().toISOString(),
        updatedAt: new Date().toISOString()
    };

    let plans = getPlans();
    plans.push(plan);
    savePlans(plans);
}
```
- 입력된 데이터를 객체로 만들어 저장
- localStorage에 저장

#### 4. 비밀번호 확인
```javascript
function verifyPassword() {
    const password = document.getElementById('verifyPassword').value;
    const plan = getPlanById(currentPlanId);

    if (plan.password !== password) {
        alert('비밀번호가 일치하지 않습니다.');
        return;
    }
    // 수정 또는 삭제 진행

```
- 저장된 비밀번호와 입력된 비밀번호 비교

#### 5. LocalStorage 관리
```javascript
function getPlans() {
    const plans = localStorage.getItem('schoolPlans');
    return plans ? JSON.parse(plans) : [];
}

function savePlans(plans) {
    localStorage.setItem('schoolPlans', JSON.stringify(plans));
}
```
- `getPlans()`: localStorage에서 데이터 읽기
- `savePlans()`: localStorage에 데이터 쓰기
- JSON 형식으로 변환하여 저장

## 데이터 저장 방식

### LocalStorage 사용

#### 장점
- 서버 없이 브라우저에 데이터 저장
- 빠른 읽기/쓰기 속도
- 무료로 사용 가능

#### 단점
- 같은 브라우저에서만 접근 가능
- 브라우저 데이터 삭제 시 함께 삭제됨
- 약 5-10MB 용량 제한

#### 저장되는 데이터 구조
```javascript
{
  "id": "unique-id-12345",
  "school": "elementary",
  "grade": "1",
  "teacherName": "홍길동",
  "subject": "국어",
  "title": "1학기 국어 수업 계획",
  "content": "수업 계획 내용...",
  "password": "1234",
  "createdAt": "2025-12-30T10:30:00.000Z",
  "updatedAt": "2025-12-30T10:30:00.000Z"
}
```

#### 데이터 확인 방법 (개발자 도구)
1. 브라우저에서 F12 키 누르기
2. "Application" 또는 "Storage" 탭 선택
3. "Local Storage" → 현재 도메인 선택
4. `schoolPlans` 키 확인

## 사용자 가이드

### 계획서 작성하기

1. **학교 선택**
   - 상단의 초등학교/중학교/고등학교 탭 클릭

2. **학년 선택**
   - 원하는 학년의 "+ 계획서 작성" 버튼 클릭

3. **정보 입력**
   - 교사 이름: 본인의 이름 입력
   - 과목: 수업 과목 입력 (예: 국어, 수학, 영어)
   - 계획서 제목: 계획서의 제목 입력
   - 계획서 내용: 상세한 수업 계획 작성
   - 비밀번호: 수정/삭제 시 사용할 비밀번호 (꼭 기억하세요!)

4. **저장**
   - "저장" 버튼 클릭
   - 저장 완료 메시지 확인

### 계획서 보기

1. **학교/학년 이동**
   - 보고 싶은 계획서가 있는 학교와 학년으로 이동

2. **계획서 클릭**
   - 계획서 목록에서 원하는 계획서 클릭

3. **내용 확인**
   - 교사 이름, 과목, 내용, 작성일 등 확인

### 계획서 수정하기

1. **계획서 열기**
   - 수정할 계획서 클릭

2. **수정 버튼 클릭**
   - "수정하기" 버튼 클릭

3. **비밀번호 입력**
   - 작성 시 입력한 비밀번호 입력
   - "확인" 버튼 클릭

4. **내용 수정**
   - 원하는 내용 수정
   - "저장" 버튼 클릭

### 계획서 삭제하기

1. **계획서 열기**
   - 삭제할 계획서 클릭

2. **삭제 버튼 클릭**
   - "삭제하기" 버튼 클릭

3. **확인**
   - 삭제 확인 메시지에서 "확인" 클릭

4. **비밀번호 입력**
   - 작성 시 입력한 비밀번호 입력
   - "확인" 버튼 클릭

## 문제 해결

### Q1: 작성한 계획서가 보이지 않아요
**A:** 다음을 확인해보세요:
- 올바른 학교 탭을 선택했나요?
- 올바른 학년을 보고 있나요?
- 브라우저의 LocalStorage가 활성화되어 있나요?
- F12 → Application → Local Storage에서 데이터를 확인해보세요

### Q2: 비밀번호를 잊어버렸어요
**A:** 안타깝게도 비밀번호는 복구할 수 없습니다. 하지만 개발자 도구로 데이터를 직접 수정할 수 있습니다:
1. F12 키를 누르고 Console 탭으로 이동
2. 다음 코드 입력:
```javascript
let plans = JSON.parse(localStorage.getItem('schoolPlans'));
plans[0].password = '새비밀번호';  // plans[0]은 첫 번째 계획서
localStorage.setItem('schoolPlans', JSON.stringify(plans));
```
3. 페이지 새로고침

### Q3: 다른 컴퓨터에서도 볼 수 있나요?
**A:** LocalStorage는 같은 브라우저에서만 접근 가능합니다. 다른 컴퓨터에서 보려면:
- 웹 호스팅 서버에 업로드하여 URL 공유
- 또는 백엔드 서버를 추가로 개발해야 합니다

### Q4: 모바일에서도 사용할 수 있나요?
**A:** 네! 반응형 디자인으로 제작되어 모바일과 태블릿에서도 사용 가능합니다.

### Q5: 데이터가 삭제될 수 있나요?
**A:** 다음 경우에 데이터가 삭제될 수 있습니다:
- 브라우저 캐시 및 쿠키 삭제 시
- 브라우저 재설치 시
- 시크릿 모드 사용 시 (창을 닫으면 삭제)

**예방 방법:**
- 주기적으로 데이터 백업 (개발자 도구에서 복사)
- 중요한 내용은 별도로 저장

## 추가 개발 아이디어

### 1. 검색 기능 추가
- 교사 이름, 과목, 제목으로 검색
- 검색 입력창과 필터링 함수 구현

### 2. 파일 첨부 기능
- 계획서에 PDF, 이미지 등 첨부
- FileReader API 사용

### 3. 인쇄 기능
- 계획서를 PDF로 저장
- window.print() 함수 활용

### 4. 데이터 내보내기/가져오기
- JSON 파일로 백업/복원
- Download 및 File Upload 기능 구현

### 5. 서버 연동
- Node.js + Express로 백엔드 구축
- MongoDB나 MySQL로 데이터베이스 연결
- 여러 컴퓨터에서 접근 가능

### 6. 사용자 인증
- 교사별 로그인 기능
- 권한 관리 (관리자, 일반 교사)
